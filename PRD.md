# 머니라함 후기 인증 메일 발송 페이지 PRD

## 1. 프로젝트 개요

머니라함 고객이 카카오톡 후기 캡처 이미지를 업로드하고 이메일 주소를 입력하면, 지정된 링크가 고객 이메일로 자동 발송되는 1페이지 웹사이트를 만든다.

초기 테스트 버전은 기능을 단순하게 유지한다. 사용자는 후기 캡처 이미지, 이메일, 개인정보 동의만 입력하고 신청한다. 관리자는 업로드된 이미지를 구글 드라이브 폴더에서 확인하고, 이메일 주소와 신청 기록은 구글 스프레드시트에서 확인한다.

## 2. 목표

- 후기 작성자가 별도 문의 없이 자료 링크를 이메일로 받을 수 있게 한다.
- 신청자의 이메일 주소를 자동으로 구글 스프레드시트에 저장한다.
- 카카오톡 후기 캡처 이미지를 구글 드라이브 폴더에 저장한다.
- 노트북과 집 로컬 PC에서 이어서 작업할 수 있도록 GitHub 업로드를 전제로 구성한다.

## 3. 비목표

- 회원가입, 로그인, 관리자 페이지는 만들지 않는다.
- 이미지 OCR, 후기 진위 자동 판별, 카카오톡 연동은 만들지 않는다.
- 결제, 쿠폰 발급, 복잡한 CRM 연동은 만들지 않는다.
- 대량 메일 발송 시스템은 만들지 않는다.

## 4. 대상 사용자

### 후기 작성 고객

- 머니라함 후기 이벤트 또는 안내를 보고 참여한다.
- 카카오톡 후기 캡처 이미지를 가지고 있다.
- 자료 링크를 받을 이메일 주소를 입력할 수 있다.

### 운영자

- 구글 드라이브와 구글 스프레드시트를 사용할 수 있다.
- 저장된 캡처 이미지와 이메일 신청 기록을 확인한다.
- 발송할 링크를 직접 지정하고 필요할 때 변경한다.

## 5. 핵심 사용자 흐름

1. 사용자가 머니라함 후기 인증 페이지에 접속한다.
2. 카카오톡 후기 캡처 이미지를 업로드한다.
3. 이메일 주소를 입력한다.
4. 개인정보 수집 및 이용 동의에 체크한다.
5. 신청 버튼을 누른다.
6. 시스템이 이미지를 구글 드라이브에 저장한다.
7. 시스템이 이메일, 신청 시간, 이미지 파일 링크, 발송 상태를 구글 스프레드시트에 저장한다.
8. 시스템이 지정된 링크를 사용자 이메일로 발송한다.
9. 화면에는 신청 완료 메시지를 보여준다.

## 6. 화면 요구사항

1페이지 랜딩 + 신청 폼 구조로 만든다.

### 상단 영역

- 브랜드명: `머니라함`
- 작은 배지: `후기 작성자 전용`
- 메인 문구: `후기를 남기고 머니라함 가이드 링크를 받아보세요`
- 보조 문구: `카카오톡 후기 캡처를 업로드하고 이메일을 입력하면, 확인용 기록을 남긴 뒤 안내 링크를 보내드립니다.`

### 안내 카드

- `01 후기 캡처 준비`
- `02 이미지 업로드`
- `03 이메일로 링크 수신`

### 신청 폼

- 파일 업로드 필드
  - 라벨: `카카오톡 후기 캡처 확인`
  - 버튼 문구: `캡처 이미지 선택하기`
  - 허용 파일: jpg, jpeg, png, webp
  - 최대 용량: 초기 테스트 기준 10MB 권장
- 이메일 입력 필드
  - placeholder: `email@example.com`
- 개인정보 동의 체크박스
  - 문구: `개인정보 수집 및 이용에 동의합니다.`
- 신청 버튼
  - 문구: `가이드 링크 메일 받기`
- 개인정보 안내 접기/펼치기
  - 수집 항목: 이메일 주소, 후기 캡처 이미지, 신청 시간
  - 이용 목적: 후기 확인 및 안내 링크 발송
  - 보관 기간: 테스트 기준 30일 또는 운영자가 정한 기간

### 완료 상태

- 성공 메시지: `신청 완료! 입력한 이메일로 가이드 링크를 보냈습니다.`
- 실패 메시지: 원인을 짧게 표시한다.
  - 이미지가 없을 때
  - 이메일 형식이 올바르지 않을 때
  - 개인정보 동의가 없을 때
  - 서버 저장 또는 메일 발송 실패

## 7. 디자인 요구사항

첨부 예시의 좌측 대형 문구 + 우측 신청 폼 구조를 참고하되, 제공된 ORYZO AI 스타일 토큰을 머니라함용으로 적용한다.

### 스타일 방향

- 다크 톤의 고급스럽고 밀도 있는 1페이지
- 배경은 `#100904`
- 본문 텍스트는 `#ffedd7`
- 주요 CTA는 `#dc5000`
- 보조 표면은 `#382416`
- 입력 필드는 투명 배경, 얇은 밝은 테두리, 각진 모서리
- 버튼은 36px pill radius
- 카드/반복 아이템은 8px 이하 또는 기존 토큰을 따르되 과하게 둥글게 만들지 않는다.

### 레이아웃

- 데스크톱: 좌측 안내, 우측 폼 2컬럼
- 모바일: 상단 안내, 하단 폼 1컬럼
- 첫 화면 안에서 신청 폼이 보이도록 구성한다.
- 페이지는 1개만 만든다.

## 8. 기능 요구사항

### 클라이언트

- 이미지 파일 선택
- 이메일 입력
- 개인정보 동의 체크
- 필수값 검증
- 제출 중 로딩 상태 표시
- 성공/실패 상태 표시

### 서버 또는 서버리스

초기 MVP는 구글 Apps Script 웹앱을 권장한다.

- 이미지 파일을 구글 드라이브 지정 폴더에 저장
- 이메일과 신청 정보를 구글 스프레드시트에 기록
- 지정된 링크를 이메일로 발송
- 처리 결과를 웹페이지에 반환

## 9. 구글 드라이브 저장 방식

무료로 시작하려면 개인 구글 계정의 구글 드라이브 폴더를 사용할 수 있다. 단, 무료 저장 공간은 Gmail, Drive, Photos를 합산해 계정 전체 용량을 사용한다.

권장 설정:

- 폴더명: `머니라함_후기캡처_업로드`
- 공유 설정: 비공개 유지
- Apps Script만 해당 폴더에 파일을 저장
- 파일명 예시: `2026-05-13_153000_user@example.com.png`

주의:

- 카카오톡 캡처에는 이름, 전화번호, 대화 내용 등 개인정보가 포함될 수 있다.
- 운영 전 개인정보 수집 동의 문구와 보관 기간을 명확히 표시해야 한다.
- 테스트 단계에서는 민감한 개인정보가 포함된 이미지를 오래 보관하지 않는 것이 좋다.

## 10. 구글 스프레드시트 저장 방식

스프레드시트명: `머니라함_후기신청_기록`

시트 컬럼:

| 컬럼 | 설명 |
| --- | --- |
| 신청시간 | 사용자가 제출한 시간 |
| 이메일 | 입력한 이메일 주소 |
| 이미지파일명 | 드라이브에 저장된 파일명 |
| 이미지URL | 드라이브 파일 링크 |
| 발송링크 | 사용자에게 보낸 링크 |
| 발송상태 | SENT 또는 FAILED |
| 오류메시지 | 실패 시 오류 내용 |

## 11. 이메일 발송 요구사항

### 메일 제목

`[머니라함] 후기 인증 가이드 링크를 보내드립니다`

### 메일 본문

- 신청 감사 문구
- 지정된 링크
- 문의 연락처 또는 카카오 채널 안내
- 자동 발송 안내

### 발송 링크

운영자가 Apps Script 설정값으로 관리한다.

예시:

`https://example.com/midam-guide`

## 12. 기술 구성

### 프론트엔드

- HTML, CSS, JavaScript로 시작
- 빌드 도구 없이도 실행 가능한 구조 권장
- 추후 필요 시 Vite 또는 Next.js로 확장 가능

### 백엔드

- 1차 MVP: Google Apps Script 웹앱
- 저장소: Google Drive
- 기록 DB: Google Sheets
- 메일: GmailApp 또는 MailApp

## 13. GitHub 고려사항

GitHub에는 웹사이트 코드와 문서만 올린다.

올리면 안 되는 것:

- 구글 Apps Script 배포 URL
- 스프레드시트 ID
- 드라이브 폴더 ID
- 개인 이메일 계정 정보
- API 키 또는 비밀번호

권장 방식:

- `.env.example`에 필요한 환경변수 이름만 적는다.
- 실제 값은 `.env` 또는 로컬 메모에만 보관한다.
- GitHub에는 `.env`를 올리지 않는다.

## 14. 성공 기준

- 사용자가 이미지와 이메일을 입력해 신청할 수 있다.
- 신청 후 이미지가 구글 드라이브에 저장된다.
- 이메일 주소와 이미지 링크가 구글 스프레드시트에 기록된다.
- 지정된 링크가 사용자 이메일로 발송된다.
- 모바일과 노트북 화면에서 폼이 깨지지 않는다.

## 15. 향후 확장 아이디어

- 관리자용 신청 목록 화면
- 중복 이메일 제출 제한
- 이미지 자동 압축
- 후기 승인 후에만 링크 발송
- Google reCAPTCHA 또는 Cloudflare Turnstile로 스팸 방지
- 발송 실패 재시도 기능
